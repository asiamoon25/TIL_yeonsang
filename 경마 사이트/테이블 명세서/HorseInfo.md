1. 1NF : 각 열의 값이 원자적이여 하며, 각 행이 유일한 키로 구분되어야 함.
2. 2NF : 1NF 를 만족하고, 모든 비기본 키 속성이 테이블의 기본 키에 완전히 함수적으로 의존해야함.
3. 3NF : 2NF 를 만족하고, 비 기본 키 속성들이 다른 비 기본 키 속성에 대해 이행적으로 의존하지 않아야함.

**HorseInfo 테이블 분할**:
- 기본 정보와 관련된 테이블.
  
- 부마와 모마 관련 정보를 별도 테이블로 분리.
  
- 경주마 관련 정보를 별도 테이블로 분리.
  
- 소유자와 조교사 관련 정보를 별도의 테이블로 분리.


1. 기본 정보 테이블(Horse)
```sql
CREATE TABLE Horses (
    hrNo VARCHAR(6) PRIMARY KEY,
    hrNm VARCHAR(30),
    hrEngNm VARCHAR(30),
    birthDt CHAR(8),
    sex CHAR(2),
    color VARCHAR(10),
    age INT,
    prdCty VARCHAR(20),
    expCty VARCHAR(20),
    breed VARCHAR(20),
    useNm VARCHAR(50)
);
```

2. 조교사 테이블(Trainers)
```sql
CREATE TABLE Trainers (
    trainerNo VARCHAR(6) PRIMARY KEY,
    trainerNm VARCHAR(20)
);
```

3. 소유자 테이블(Owners)
```sql
CREATE TABLE Owners (
    ownerNo VARCHAR(6) PRIMARY KEY,
    ownerNm VARCHAR(40)
);
```

4. 부마 및 모마 정보 테이블 (ParentHorses)
```sql
CREATE TABLE ParentHorses (
    horseNo VARCHAR(6),
    parentType VARCHAR(10), -- 'Sire' or 'Dam'
    parentHorseNo VARCHAR(6),
    parentHorseNm VARCHAR(30),
    country CHAR(3),
    birthYear CHAR(4),
    FOREIGN KEY (horseNo) REFERENCES Horses(hrNo)
);
```

5. 경주 기록 테이블(RacingRecords)
```sql
CREATE TABLE RacingRecords (
    recordId INT AUTO_INCREMENT PRIMARY KEY,
    hrNo VARCHAR(6),
    eventDate CHAR(8),
    eventPlace VARCHAR(40),
    eventResult VARCHAR(10),
    FOREIGN KEY (hrNo) REFERENCES Horses(hrNo)
);
```
