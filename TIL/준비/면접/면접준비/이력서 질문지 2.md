
* TCP vs UDP
	* 동기와 비동기 느낌
	* TCP
		* 연결 지향
		* 연결을 받았다는 수신 확인(ACK) 를 받음.
		* 데이터의 정확성과 순서가 중요한 곳에 사용
	* UDP
		* TCP 보다 빠르지만 신뢰성이 떨어짐.
		* 속도와 효율성이 더 중요한 경우에 적합.

* TCP 3,4 way handshake
	* 3 way handshake
		* 상대방 컴퓨터와 사전에 세션을 수립하는 과정
			* Client -> Server : TCP SYN
			* Server -> Client : TCP SYN ACK
			* Client -> Server : TCP ACK
